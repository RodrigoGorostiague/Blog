Descripción de la Arquitectura de la Aplicación y del Proyecto:

Comenzamos un fascinante viaje al construir nuestra aplicación desde cero, decidi reciclar piezas y crear mi propio servidor en lugar de depender de plataformas de despliegue convencionales. Esta elección me ha permitido estar involucrado de principio a fin, desde la selección del hardware hasta la implementación de la arquitectura de contenedores y la automatización del ciclo de vida de la aplicación.

Backend con NestJS:
La lógica de negocio y el manejo de datos están centralizados en una API RESTful construida con NestJS, un framework de desarrollo para Node.js. Este componente proporciona servicios robustos y escalables, implementados en un contenedor Docker independiente.

Frontend con Angular:
La interfaz de usuario se desarrolla utilizando Angular, un framework de desarrollo de aplicaciones web. La aplicación frontend se encuentra en un contenedor Docker separado para garantizar su independencia y escalabilidad.

Persistencia de Datos con PostgreSQL:
La base de datos de la aplicación se implementa utilizando PostgreSQL, un sistema de gestión de bases de datos relacional. Para garantizar la persistencia de los datos, hemos configurado un volumen externo en el host para almacenar los datos de PostgreSQL fuera del contenedor.
